Include the English library file in a Ruby script, and you can reference the global variables such as $_ using less cryptic names, listed below.

Without 'English':

$\ = ' -- '
"waterbuffalo" =~ /buff/
print $', $$, "\n"

With English:

require "English"

$OUTPUT_FIELD_SEPARATOR = ' -- '
"waterbuffalo" =~ /buff/
print $POSTMATCH, $PID, "\n"
